# Yandex Metrika Counter Monitoring

Проект для мониторинга поступления данных по счетчикам Яндекс Метрики.
Если счетчик "отваливается" и объем данных в последнем завершенном временном слоте
сильно ниже обычного уровня, скрипт формирует email-алерт.

## Состав проекта

- `metrika_monitor.py` — логика проверки счетчиков и отправки письма
- `ym_counters_monitoring.py` — DAG для Airflow

## Как работает

1. Берется последний завершенный временной слот
2. Для каждого счетчика запрашивается текущее значение
3. Рассчитывается baseline по историческим слотам
4. Если текущее значение сильно ниже baseline, формируется алерт
5. Если отвалилось несколько счетчиков, отправляется одно общее письмо

## Переменные окружения

Создайте файл `.env` в корне проекта и укажите в нем следующие переменные:

```env
YM_OAUTH_TOKEN=

YM_COUNTER_1=
YM_COUNTER_2=
YM_COUNTER_3=
YM_COUNTER_4=
YM_COUNTER_5=

SMTP_LOGIN=
SMTP_PASSWORD=
EMAIL_TO=

Где:

YM_OAUTH_TOKEN — OAuth-токен для API Яндекс Метрики
YM_COUNTER_1 ... YM_COUNTER_5 — ID счетчиков, которые нужно мониторить
SMTP_LOGIN — логин почты отправителя
SMTP_PASSWORD — пароль или пароль приложения для почты
EMAIL_TO — почта получателя уведомлений
